# Rap Loop（快速应用原型循环）

## 核心原则
未满足完成条件不得结束任务。

## 循环流程

```
Plan（计划）→ Act（执行）→ Verify（验证）→ Repeat（重复）
```

### 1. Plan（计划）
- 明确当前要实现的功能
- 拆分为可执行的小步骤
- 记录到 TASKS.md

### 2. Act（执行）
- 按步骤编写代码
- 遵循 DEVELOPMENT_RULES.md
- 实时更新 TASKS.md 状态

### 3. Verify（验证）
- 必须真实运行项目
- 必须检查控制台报错
- 必须验证功能真实可用
- 遵循 TESTING_RULES.md

### 4. Repeat（重复）
- 如果验证未通过 → 回到 Plan 或 Act
- 如果验证通过 → 标记完成，进入下一个任务
- 直到所有任务完成

## 完成条件（Definition of Done）
- [ ] 代码已编写
- [ ] 项目已真实运行
- [ ] 控制台无报错
- [ ] 功能真实可用
- [ ] TASKS.md 状态已更新
- [ ] README 已同步（如有需要）
- [ ] 重要决策已记录

## 禁止事项
- 禁止在未验证的情况下标记任务完成
- 禁止跳过验证步骤
- 禁止忽略控制台报错
