# Brainstorming Rules

## 核心原则
所有开发前必须先讨论需求。不允许直接开始写代码。

## 必须明确的内容

### 1. 用户 (Who)
- 目标用户是谁？
- 用户的使用场景是什么？

### 2. 功能 (What)
- 核心功能列表
- 功能的优先级
- 边界条件和异常情况

### 3. 数据结构 (Data)
- 需要哪些数据模型？
- 字段定义和类型
- 数据之间的关系

### 4. 页面结构 (UI)
- 页面路由设计
- 组件树结构
- 状态管理方案

## 执行规则
- 未完成 Brainstorming 不得进入开发阶段
- 需求和设计必须记录到 PROJECT_PLAN.md
- 如有不明确的地方，必须先澄清再动手
